Enrollment Survey

For people considering participating in the course, we would greatly appreciate ("require") you completing the short Google Docs survey. This will help us determine how large a room we will need, what background people have, etc.

This is an informal, group study style course. The goal is to assemble an active group doing Data Science problems at UCD. We hope to attract people (graduate students, postdocs, faculty, and good undergraduates) from around the different departments, etc. on campus who are interested in

  • analyzing interesting, complex and/or large data,
  • gaining experience using modern data technologies,
  • learning new statistical/machine learning approaches and methods,
  • implementing computational solutions and data analyses, and
  • creating compelling and insightful visualizations of information.

    Structure of the Explorations

    We plan to focus the activities around two or more specific data analysis problems. We will first introduce the sources of data that we will be using and the general questions that are of interest, together with the important issues and possible approaches to answering those questions. Throughout each week we will work in groups on exploring the data and addressing the questions, discussing issues on Piazza as they arise. During each weekly meeting we will discuss some of the more complex issues and introduce new statistical or computational techniques as they become appropriate. These might include topics such as neural networks, deep learning and boosting on the statistical side, MapReduce/Hadoop for parallel computing, or a particular R package or Java library on the computational side.

    At the end of the quarter, groups will present their results. As extra incentive, we will give prizes to the best approaches in a variety of categories (e.g., best visualization, best interactive application, best predictive performance).

    We strongly encourage people who are facing Data Sciences challenges in their research to come and present these to the group, hopefully to educate each other, get suggestions as to how to solve some of the challenges, and explore possibilities for accessing different sources of data (e.g. Web scraping, APIs) or managing data. We would love to hear about these and help out as is feasible.

    Topics

    The focus is on authentic data analysis and statistical reasoning. Therefore, the focus is less on learning a predetermined set of statistical techniques or computing technologies and instead on using the most appropriate tools to address a given problem. However, some of the topics we will use and/or introduce may include
    • parallel computing (with any of: R, Python, MapReduce, Hadoop, C, GPUs)
    • statistical/machine learning techniques (e.g. boosting, bagging, classification trees, random forests, logistic regression, neural networks, deep learning).
    • visualization (static and interactive Web-based visualization)

    Prerequisites/Background

    In short, we want people who are motivated, interested, have initiative and some independence and who want to learn more Data Science skills. You should
    • have reasonable programming skills in a language such as R, Python, MATLAB, Java, C, C++ (it does not matter which language).
    • have sufficient knowledge of statistics to understand heuristic descriptions of new methods (at the level of Introduction to Statistical Learning with Applications in R) if not the full technical details.
    • be willing to participate, speak up, explore data and write code.
    If you would like to participate but are unsure whether you have the appropriate programming and/or statistical skills then please feel free to email Duncan or Paul directly.

    Examples Data Problems

    The datasets to be analyzed are still being finalized, but below are some illustrative examples. If you know of a publicly available dataset that you think would be interest then, again, please feel free to email Duncan or Paul with details.
  • Predicting race performance for runners using historical race results and (if available) training data.
    This involves data matching (across races) and data fusion (across data sources) as well as flexible predictive statistical techniques.
  • Image classification: classify each of a collection of pictures as either containing a dog or a cat.
    This task is very easy for humans to do, but is surprisingly challenging to automate. Approaches will involve some image processing as well as statistical/machine learning techniques for classification.

    • Course Learning Goals

    • Enhance skills in how to map a question of interest into a statistical framework and to identify and learn appropriate methods for estimation and prediction.
    • Gain experience with computational techniques to explore, visualize and fit models with data.

    Relevant Books

    • Introduction to Statistical Learning with Applications in R Gareth James, Daniela Witten, Trevor Hastie, Rob Tibshirani
    • Elements of Statistical Learning Trevor Hastie, Rob Tibshirani and Jerry Friedman
    • Pattern Recognition and Machine Learning, Christopher Bishop
    • Machine Learning: A Probabilistic Approach, Kevin Murphy
